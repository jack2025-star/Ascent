###  JavaScript：基础语法核心笔记

| **核心概念 / 术语** | **详细说明与代码示例** |
| :--- | :--- |
| **变量声明**(let / const / var) | **1. let**：可重新赋值，具有**块级作用域**。**2. const**：常量，声明后**不可重新赋值**，必须初始化。具有块级作用域（**现代开发首选**）。**3. var**：旧语法，**函数级作用域**，存在变量提升，**不推荐使用**。```javascript<br>// 推荐写法<br>let age = 25;<br>age = 26; //  允许<br><br>const PI = 3.14;<br>// PI = 3.15; //  报错：Assignment to constant variable.<br><br>// 不推荐<br>var old = "过时"; // ️ 避免使用<br>``` |
| **数据类型**(Data Types) | JS是弱类型语言，主要包含：- **string**：文本，如 `"Hello"`- **number**：整数或浮点数，如 `30`, `3.14`- **boolean**：逻辑值 `true` / `false`- **undefined**：声明但未赋值- **null**：故意赋予的空值**typeof 运算符**：用于检查数据类型。```javascript<br>let name = "Jack";<br>console.log(typeof name); // "string"<br>console.log(typeof 42);   // "number"<br>console.log(typeof null); // "object" (历史遗留Bug)<br>``` |
| **相等运算符对比**(== vs ===) | - **== (宽松相等)**：比较前会进行**隐式类型转换**，只看值是否相等。- **=== (严格相等)**：**不会**转换类型，要求值和类型都相同（**强烈推荐**）。```javascript<br>console.log(5 == "5");  // true (字符串转数字)<br>console.log(5 === "5"); // false (类型不同)<br>``` |
| **字符串与模板字面量** | - **传统拼接**：使用 `+` 号，繁琐且易错。- **模板字面量**：使用**反引号 (`)** 和 `${变量}`，支持换行和嵌入表达式。<br><br>```javascript<br>let user = "Alice";<br>// 传统<br>let msg1 = "Hello, " + user;<br><br>// 模板字面量 (推荐)<br>let msg2 = `Hello, ${user}`; <br>``` |
| **Truthy 和 Falsy** | JS中所有值都能被隐式转换为布尔值：**5个 Falsy (假值)**：`0`, `""` (空字符串), `undefined`, `null`, `NaN`**Truthy (真值)**：除上述5个外，其余均为真（包括 `"0"`, `[]`, `{}`）。 |
| **逻辑运算符** | - **&& (AND)**：全真才真。常用于“如果...则...”逻辑。- **\|\| (OR)**：有真就真。常用于设置默认值。- **! (NOT)**：取反。```javascript<br>let a = true, b = false;<br>console.log(a && b); // false<br>console.log(a || b); // true<br>console.log(!a);     // false<br>``` |
| **类型转换** | - **显式转换**：手动调用函数。  `Number("123")` → 123  `String(123)` → "123"  `Boolean(0)` → false- **隐式转换**：JS引擎自动转换（如比较运算时），容易产生Bug。 |

| 特性 | **var** | **let** | **const** |

### ️ 常见错误与注意事项

1.  **混淆 `=` 和 `===`**：`=` 是赋值，`==` 是宽松比较，`===` 是严格比较。写条件判断时务必使用 `===`。
2.  **const 的误区**：`const` 保证的是**引用的地址不变**。如果 `const` 声明的是对象或数组，其内部的属性或内容是可以修改的。
    ```javascript
    const arr = [1, 2];
    arr.push(3); //  允许修改内容
    // arr = []; //  报错：不能改变引用地址
    ```
3.  **Falsy 值陷阱**：注意数字 `0` 和空字符串 `""` 在 `if` 判断中会被视为 `false`。
4.  **NaN 的特性**：`NaN` (Not a Number) 是 falsy 值，且 `NaN === NaN` 的结果是 `false`。

###  本节总结

本章是JavaScript的基石。核心在于理解**变量声明的规范**（优先用 `const`，变动用 `let`，弃用 `var`）以及**数据类型的分类**。在编写逻辑时，要熟练掌握**模板字面量**来简化字符串操作，并养成使用**严格相等 (`===`)** 的习惯以避免类型转换带来的隐式Bug。同时，理解 **Truthy/Falsy** 机制对于编写简洁的条件判断（如 `if(user)`）至关重要。
