# 项目操作日志

## 📅 2026-05-16：静态文件服务配置更新

### 🎯 任务目标
实现通过 `http://localhost:3000` 直接访问登录页面，替代手动打开本地 HTML 文件的方式，并解决潜在的跨域问题。

### 🛠️ 核心变更步骤
1. **引入路径模块**：在 `src/app.js` 中引入 Node.js 内置的 `path` 模块，用于处理跨平台文件路径。
2. **启用静态托管**：配置 `express.static(path.join(__dirname))`，将 `src` 目录映射为服务器的公开资源根目录。
3. **统一入口路由**：修改首页路由，使用 `res.sendFile()` 显式返回 `login.html`，确保根路径访问即渲染登录界面。

### 💡 宏观原理说明
*   **统一端口与同源策略**：通过将前端页面托管在后端服务器上，使前后端运行在同一端口（3000）下。这不仅简化了访问流程，更从根本上消除了浏览器同源策略导致的跨域拦截。
*   **路径兼容性保障**：利用 `path.join(__dirname)` 动态拼接绝对路径，确保代码在 Windows（反斜杠）和 Unix/Linux（正斜杠）系统下均能正确定位文件。
*   **自动化资源分发**：`express.static` 中间件接管了静态资源的查找工作，浏览器请求 `/login.html` 时，服务器会自动从指定目录读取并返回文件内容，无需编写冗余的文件读取逻辑。

### ⚠️ 维护与测试注意事项
*   **进程重启**：Node.js 是单线程应用，修改 `app.js` 后必须终止旧进程并重新启动，新配置方可生效。
*   **资源位置约束**：当前静态资源托管范围限定在 `src` 目录。若后续新增 CSS 或 JS 独立文件，需将其移入该目录才能被浏览器正常加载。
*   **缓存干扰排除**：浏览器可能缓存旧版本的 HTML 或样式文件。测试时建议使用 `Ctrl + F5` 强制刷新，或在开发者工具中禁用缓存。
